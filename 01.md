Microsoft Agent Framework로 AI 에이전트를 개발하기 위한 환경을 구축한다.  
Microsoft Agent Framework 관련 문서를 보면 LLM은 Azure를 사용하는 경우가 많아서 Azure만 사용할 수 있는 것으로 착각하기 쉬운데 절대 **Azure만 사용하는 것이 아니다** OpenRouter 또는 Poe의 OpenAI 호환 API 등을 사용할 수 있다.

## 1.2 LLM API 키 준비
**OpenRouter** 또는 **Poe** 중 하나를 선택한다.

### OpenRouter (권장)
OpenRouter는 Claude, GPT-4o, Gemini 등 다양한 모델을 하나의 API로 사용할 수 있는 플랫폼이다.

**API 키 발급:**
1. https://openrouter.ai 에 가입한다
2. **API Keys** 메뉴에서 키를 생성한다
3. 사용할 모델: `anthropic/claude-sonnet-4-5`

**지원 모델 예시:**

| 모델 ID | 설명 |
|---------|------|
| `anthropic/claude-sonnet-4-5` | Claude Sonnet - 균형잡힌 성능 (권장) |
| `anthropic/claude-haiku-4-5` | Claude Haiku - 빠르고 경제적 |
| `openai/gpt-4o` | GPT-4o |
| `google/gemini-2.0-flash` | Gemini 2.0 Flash |

### Poe (대안)

1. https://poe.com 에 로그인한다
2. **Settings → API Key** 에서 키를 생성한다
3. Base URL: `https://api.poe.com/llm/v1`

## 1.3 환경 변수 설정
모든 예제는 아래 세 가지 환경 변수를 사용한다. Azure 관련 변수는 사용하지 않는다.

| 변수명 | 설명 |
|--------|------|
| `LLM_API_KEY` | API 키 (필수) |
| `LLM_BASE_URL` | 엔드포인트 URL |
| `LLM_MODEL` | 사용할 모델 이름 |

**Windows (PowerShell):**

```powershell
# OpenRouter
$env:LLM_API_KEY="sk-or-your-key-here"
$env:LLM_BASE_URL="https://openrouter.ai/api/v1"
$env:LLM_MODEL="anthropic/claude-sonnet-4-5"

# 영구 설정
[System.Environment]::SetEnvironmentVariable('LLM_API_KEY', 'sk-or-your-key', 'User')
[System.Environment]::SetEnvironmentVariable('LLM_BASE_URL', 'https://openrouter.ai/api/v1', 'User')
[System.Environment]::SetEnvironmentVariable('LLM_MODEL', 'anthropic/claude-sonnet-4-5', 'User')
```

**Mac/Linux:**

```bash
export LLM_API_KEY="sk-or-your-key-here"
export LLM_BASE_URL="https://openrouter.ai/api/v1"
export LLM_MODEL="anthropic/claude-sonnet-4-5"

# 영구 설정 (~/.bashrc 또는 ~/.zshrc에 추가)
echo 'export LLM_API_KEY="sk-or-your-key-here"' >> ~/.bashrc
echo 'export LLM_BASE_URL="https://openrouter.ai/api/v1"' >> ~/.bashrc
echo 'export LLM_MODEL="anthropic/claude-sonnet-4-5"' >> ~/.bashrc
source ~/.bashrc
```

**Poe를 사용하는 경우:**

```powershell
$env:LLM_API_KEY="your-poe-api-key"
$env:LLM_BASE_URL="https://api.poe.com"
$env:LLM_MODEL="claude-sonnet-4-20250514"
```

## 1.4 프로젝트 초기 설정

### 프로젝트 생성

```bash
dotnet new console -o MyFirstAgent
cd MyFirstAgent
```

### 필수 패키지 설치

```bash
# Microsoft Agent Framework (OpenAI 호환)
dotnet add package Microsoft.Agents.AI
dotnet add package Microsoft.Agents.AI.OpenAI

# 환경 변수 관리
dotnet add package DotNetEnv
```

생성된 `MyFirstAgent.csproj`:

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net9.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.Agents.AI" Version="*" />
    <PackageReference Include="Microsoft.Agents.AI.OpenAI" Version="*" />
    <PackageReference Include="DotNetEnv" Version="3.1.1" />
  </ItemGroup>

</Project>
```

## 환경 확인 테스트

`Program.cs`에 다음 코드를 작성하고 실행한다:

```csharp
using DotNetEnv;

// .env 파일 로드
Env.Load();

var apiKey  = Environment.GetEnvironmentVariable("LLM_API_KEY");
var baseUrl = Environment.GetEnvironmentVariable("LLM_BASE_URL") 
              ?? "https://openrouter.ai/api/v1";
var model   = Environment.GetEnvironmentVariable("LLM_MODEL") 
              ?? "anthropic/claude-sonnet-4-5";

if (string.IsNullOrEmpty(apiKey))
{
    Console.WriteLine("❌ LLM_API_KEY 환경 변수가 설정되지 않았다.");
    Console.WriteLine("   .env 파일 또는 시스템 환경 변수를 확인하라.");
}
else
{
    Console.WriteLine("✅ API 키 로드 성공");
    Console.WriteLine($"   키 앞 8자리: {apiKey[..Math.Min(8, apiKey.Length)]}...");
    Console.WriteLine($"✅ Base URL: {baseUrl}");
    Console.WriteLine($"✅ 모델: {model}");
    Console.WriteLine($"✅ .NET 버전: {Environment.Version}");
    Console.WriteLine("환경 준비 완료!");
}
```

```bash
dotnet run
# ✅ API 키 로드 성공
# ✅ Base URL: https://openrouter.ai/api/v1
# ✅ 모델: anthropic/claude-sonnet-4-5
# ✅ .NET 버전: 9.0.x
# 환경 준비 완료!
```
