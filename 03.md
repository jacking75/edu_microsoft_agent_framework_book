## 3.1 AIAgentBuilder 패턴

### 왜 AIAgentBuilder인가?

LLM 제공자가 바뀔 때마다 코드 전체를 수정하면 유지보수가 어렵다. `AIAgentBuilder`는 이 문제를 해결하는 추상화 레이어다.

```
OpenRouter ─┐
            ├─ AIAgentBuilder ─── Build() ──→ AIAgent ──→ RunAsync()
Poe ────────┘
```

`Build()` 이후의 코드는 LLM이 무엇이든 동일하다. 제공자를 바꾸려면 `WithOpenRouter()` → `WithPoe()` 한 줄만 수정하면 된다.

### AIAgentBuilder 사용법

```csharp
// OpenRouter 사용
var agent = AIAgentBuilder
    .WithOpenRouter(apiKey, "anthropic/claude-sonnet-4-5")
    .Build("MyAgent", "당신은 도움이 되는 어시스턴트다.");

// Poe 사용 (동일한 Build() 코드)
var agent = AIAgentBuilder
    .WithPoe(apiKey, "claude-sonnet-4-20250514")
    .Build("MyAgent", "당신은 도움이 되는 어시스턴트다.");

// 환경 변수로 자동 설정 (권장 - 코드 변경 없이 .env만 수정)
var agent = AIAgentBuilder
    .FromEnvironment()
    .Build("MyAgent", "당신은 도움이 되는 어시스턴트다.");
```

## 3.2 기본 Agent 생성 및 실행

### 프로젝트 준비

```bash
dotnet new console -o FirstAgent
cd FirstAgent
dotnet add package Microsoft.Agents.AI
dotnet add package Microsoft.Agents.AI.OpenAI
dotnet add package DotNetEnv
```

### .env 파일 생성

```bash
# .env (프로젝트 루트)
LLM_API_KEY=sk-or-your-openrouter-key
LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL=anthropic/claude-sonnet-4-5
```

`.gitignore`에 추가한다:

```
.env
*.env
bin/
obj/
```

### Program.cs 작성

```csharp
using DotNetEnv;
using AgentFrameworkBook.Shared;

// .env 파일 로드
Env.Load();

Console.WriteLine("🤖 첫 번째 Agent 시작");
Console.WriteLine();

// AIAgentBuilder로 에이전트 생성 (LLM 제공자를 환경 변수로 결정)
var agent = AIAgentBuilder
    .FromEnvironment()
    .Build(
        name: "HelloAgent",
        instructions: "당신은 친절하고 도움이 되는 AI 어시스턴트다. 사용자의 질문에 명확하고 간결하게 답변한다."
    );

// 단일 질문 실행
var result = await agent.RunAsync("안녕하세요! 당신은 누구인가요?");

Console.WriteLine("💬 사용자: 안녕하세요! 당신은 누구인가요?");
Console.WriteLine();
Console.WriteLine("🤖 Agent: " + result.Text);
Console.WriteLine();
Console.WriteLine("✅ Agent 실행 완료!");
```

### 실행

```bash
dotnet run
```

**예상 출력:**

```
🤖 첫 번째 Agent 시작

💬 사용자: 안녕하세요! 당신은 누구인가요?

🤖 Agent: 안녕하세요! 저는 AI 어시스턴트입니다. 질문에 답하고 다양한 작업을 도와드립니다. 무엇을 도와드릴까요?

✅ Agent 실행 완료!
```

## 3.3 대화형 Agent 프로그램

### 멀티턴 대화 구현

단일 질문이 아니라 연속적인 대화가 가능한 프로그램을 만든다.

```csharp
using DotNetEnv;
using AgentFrameworkBook.Shared;

Env.Load();

Console.WriteLine("🤖 대화형 Agent 시작 (종료: 'quit' 입력)");
Console.WriteLine("────────────────────────────────────────");
Console.WriteLine();

var agent = AIAgentBuilder
    .FromEnvironment()
    .Build(
        name: "ChatAgent",
        instructions: @"당신은 친절하고 도움이 되는 AI 어시스턴트다.
사용자와 자연스럽게 대화한다.
답변은 간결하고 명확하게 한다."
    );

// Thread를 사용하여 대화 컨텍스트 유지
var thread = agent.CreateThread();

while (true)
{
    Console.Write("💬 사용자: ");
    var input = Console.ReadLine();

    if (string.IsNullOrWhiteSpace(input)) continue;
    if (input.Trim().ToLower() == "quit") break;

    Console.WriteLine();
    Console.Write("🤖 Agent: ");

    var result = await agent.RunAsync(input, thread);
    Console.WriteLine(result.Text);
    Console.WriteLine();
}

Console.WriteLine("대화를 종료한다.");
```

### RunAsync vs RunStreamAsync

**`RunAsync`:** 응답이 완성된 후 전체를 반환한다. 짧은 응답에 적합하다.

```csharp
var result = await agent.RunAsync("질문");
Console.WriteLine(result.Text);
```

**`RunStreamAsync`:** 응답을 스트림으로 실시간 출력한다. 긴 응답에 적합하다.

```csharp
await foreach (var chunk in agent.RunStreamAsync("질문"))
{
    Console.Write(chunk.Text);
}
Console.WriteLine();
```

## 3.4 설정 관리

### 환경 변수 우선순위

```
시스템 환경 변수 (최우선)
    ↓
.env 파일
    ↓
기본값
```

### AIAgentBuilder의 설정 정보 확인

```csharp
var builder = AIAgentBuilder.FromEnvironment();
builder.PrintConfig();  // API Key 마스킹하여 출력
```

## 3.5 에러 처리

### 일반적인 오류와 해결책

```csharp
try
{
    var agent = AIAgentBuilder.FromEnvironment().Build("Agent", "...");
    var result = await agent.RunAsync("질문");
    Console.WriteLine(result.Text);
}
catch (InvalidOperationException ex) when (ex.Message.Contains("LLM_API_KEY"))
{
    Console.WriteLine("❌ API 키가 설정되지 않았다.");
    Console.WriteLine("   .env 파일에 LLM_API_KEY를 설정하라.");
}
catch (HttpRequestException ex)
{
    Console.WriteLine($"❌ 네트워크 오류: {ex.Message}");
    Console.WriteLine("   LLM_BASE_URL과 인터넷 연결을 확인하라.");
}
catch (Exception ex)
{
    Console.WriteLine($"❌ 오류: {ex.Message}");
}
```

**일반적인 오류:**

| 오류 | 원인 | 해결책 |
|------|------|--------|
| `InvalidOperationException: LLM_API_KEY` | API 키 미설정 | `.env`에 `LLM_API_KEY` 추가 |
| `HttpRequestException 401` | API 키 잘못됨 | 키를 다시 확인 |
| `HttpRequestException 429` | Rate Limit 초과 | 잠시 후 재시도 |
| `HttpRequestException 404` | 잘못된 Base URL | `LLM_BASE_URL` 확인 |
